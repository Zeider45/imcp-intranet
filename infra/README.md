# Infra y Convenciones para la rama dev

Convenciones propuestas:
- Branch para desarrollo principal: dev (ya creada).
- Nombres de módulos en carpeta modules/<slug>.
- Cada módulo contiene README.md para especificación inicial y .gitkeep para crear la carpeta.
- Labels sugeridos (en repo): enhancement, module, priority:high/medium/low
- Pull Requests: abrir PR hacia main desde dev con descripción y lista de módulos añadidos.
